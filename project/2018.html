# 战略眼光

几年前（14年）阿里前端团队，借助nodejs提出一个新的更完善的==前后端分离==的体系，即基于NodeJS“全栈”式开发。 现在阿里起了个名词叫：BFF = Backend For Frontend（服务于前端的后端）

如图

# 旧体系

后端 = 服务器 前端 = 浏览器端

最具体的表现就是我们常用的SPA应用：

  1. 渲染，取值都在客户端进行，有性能的问题
  2. 需要等待资源到齐才能进行，会有短暂白屏与闪动
  3. 在移动设备低速网路的体验奇差无比
  4. 渲染都在客户端，模版无法重用，SEO实现 麻烦

# 新体系

解决痛点：

  1. 服务器端渲染，速度更快
  2. 路由共享、验证共享、模板共享
  3. 跨终端推送页面
  4. 前端可提前开发，node层可以让测试做mock数据
  5. seo容易
  6. 可用更多的方法优化性能 Comet、Bigpipe等
  7. 权限层控制在BFF层、使下层服务更加独立和纯粹
  8. 应用缓存 BFF层
  9. 第三方调用在 BFF层
  10. 其实是可以灵活控制 服务端渲染还是浏览器渲染；进而也可以根据不同终端提供不同API，也可以像不同终端推送不同的适配页面。

BFF的思维，可以让前端开发模式有更好的变化 大前端架构思考与选择
