# 网站搭建步骤

WARNING

官方提示：Node.js版本 >= 8

# 搭建步骤(yarn安装)

  1. 现有项目安装
# 将 VuePress 作为一个本地依赖安装
yarn add -D vuepress # 或者：npm install -D vuepress

# 新建一个 docs 文件夹
mkdir docs

# 新建一个 markdown 文件
echo '# Hello VuePress!' > docs/README.md

# 开始写作
npx vuepress dev docs

  1. 配置 可在package.json里加一些脚本：
{
  "scripts": {
    "docs:dev": "vuepress dev docs", // 本地开发
    "docs:build": "vuepress build docs", // 编译打包
    "deploy": "bash deploy.sh" // 运行编译打包、部署上线脚本
  }
}

deploy.sh脚本文件，放在根目录下

#!/usr/bin/env sh 

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
yarn docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名，请设置；
# 因为每次部署后，github项目的Settings中Custom domain会重置，所以建议有域名的话，设置此CNAME
# echo 'xxx.com' > CNAME

git init
git add -A
git commit -m 'comments'

# 下边两种发布选其一，否则无法推送代码到github

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:kindredPP/blog.git master

cd -

  1. 部署

每次提交前 修改 deploy.sh脚本中的comments

运行 yarn deploy脚本即可

  1. 域名绑定

首先我们要购买域名：

以阿里云为例，购买过程就不赘述了，找到域名解析菜单

  • 4.1 添加@的A记录：指向你的网页的IP地址，ip地址可以通过ping https://yourID.github.io/yourREPO 得到
  • 4.2 添加www的CNAME记录：填写域名yourID.github.io
  1. 高级用法（参考官网）

  • 5.1 配置高级的sidebar

    官网单独讲了侧边栏分组和多个侧边栏，我们可以将这两种结合起来，让侧边栏既能分组情况下展示多个:

    首先项目目录需要设置如下：

        .
    ├─ README.md
    ├─ foo/
      ├─ README.md
      ├─ one.md
      ├─ two.md
      ├─ three.md
      └─ four.md

    config.js文件修改

      // 修改 /docs/.vuepress/config.js配置
  module.exports = {
    themeConfig: {
      sidebar: {
        // 与目录名匹配
        '/foo/': [
          {
            title: 'group 1',// group name
            children: [
              '/',     // foo/README.md
              '/foo/one',  // foo/one.md
              '/foo/two'   // foo/two.md
            ],
          },
          {
            title: 'group 2',// group name
            children: [
              '/foo/three',  // foo/three.md
              '/foo/four'    // foo/four.md
            ],
          }
        ],
      }
    }
